# 📊 JK BMS (JK-PB Series) Modbus RTU Register Map

เอกสารนี้สรุปข้อมูลตําแหน่ง **Register** ทั้งหมดของบอร์ด **JK BMS (ซีรีส์ PB)** ที่ดึงมาแสดงผลบน Dashboard หน้าเว็บ โดยอธิบายความสัมพันธ์ระหว่าง **Register Address** ดั้งเดิม และ **ตำแหน่ง Index จริง** ในคำสั่งอ่านแบบบีบอัด (Gaps-Free Modbus)

---

## ⚠️ ข้อมูลสำคัญที่สุดเรื่องโปรโตคอล JK BMS Modbus RTU
บอร์ด JK BMS ซีรีส์ PB เมื่อมีคำสั่งขออ่านข้อมูลแบบกลุ่ม (Read Multiple Registers - ฟังก์ชัน `03`) **จะบีบอัดข้อมูลส่งกลับมาโดยตัดช่องว่าง (Register ที่ไม่ได้ใช้งาน) ออกทั้งหมด** 

ส่งผลให้ตำแหน่งการดึงค่า (Array Index) ในผลลัพธ์การอ่าน ดึงต่อเรียงกันโดยตรง ไม่สอดคล้องกับ Register Offset แบบโปรโตคอล Modbus ทั่วไป

---

## 1. 🔋 แรงดันไฟฟ้าแต่ละเซลล์ (Cell Voltages)
* **Register เริ่มต้น:** `0x1200`
* **จำนวนที่อ่าน:** 20 Registers (`0x1200` - `0x1213`)
* **โครงสร้างข้อมูล:** แบบ 16-bit Unsigned Integer (`U16`)

| ตําแหน่งดึงค่า (Index) | Register | ค่าที่อ่านได้ | หน่วยวัด | การคำนวณในโค้ด |
| :---: | :---: | :--- | :---: | :--- |
| `cellRegs[0]` | `0x1200` | แรงดันไฟฟ้าเซลล์ที่ 1 | mV | `mV / 1000` -> (V) |
| `cellRegs[1]` | `0x1201` | แรงดันไฟฟ้าเซลล์ที่ 2 | mV | `mV / 1000` -> (V) |
| `...` | `...` | `...` | `...` | `...` |
| `cellRegs[15]` | `0x120F` | แรงดันไฟฟ้าเซลล์ที่ 16 | mV | `mV / 1000` -> (V) |

## 2. ⚡ ความต้านทานขั้วสาย (Cell Wire Resistance)
* **Register เริ่มต้น:** `0x1240`
* **จำนวนที่อ่าน:** 24 Registers
* **โครงสร้างข้อมูล:** แบบ 16-bit Unsigned Integer (`U16`)

| ตําแหน่งดึงค่า (Index) | Register | ค่าที่อ่านได้ | หน่วยวัด | การคำนวณในโค้ด |
| :---: | :---: | :--- | :---: | :--- |
| `statRegs[0]` | `0x1240` | ผลต่างแรงดันเซลล์ต่ำสุด (Volt. Diff Min) | mV | ค่าตรง |
| `statRegs[1]` | `0x1241` | ผลต่างแรงดันเซลล์สูงสุด (Volt. Diff Max) | mV | ค่าตรง |
| `statRegs[2]` | `0x1242` | แรงดันไฟฟ้าเฉลี่ยเซลล์ (Cell AVG) | mV | `mV / 1000` -> (V) |
| `statRegs[3]` | `0x1243` | จำนวนเซลล์ที่เชื่อมต่อจริง | เซลล์ | ค่าตรง |
| `statRegs[5 + i]` | `0x1245 + i` | ความต้านทานขั้วสายเซลล์ที่ `i` | mΩ x 1000 | `rawR / 1000` -> (mΩ) |

*ตัวอย่าง: ค่าความต้านทานขั้วสายอ่านจาก Modbus ได้ 198 จะถูกคำนวณเป็น $198 / 1000 = 0.198\text{ mΩ}$*

| ตําแหน่งดึงค่า (Index) | Register | ค่าที่อ่านได้ | หน่วยวัด | การคำนวณในโค้ด |

| `rtRegs1[15]` | `0x128F` | อุณหภูมิแบตเตอรี่ จุดที่ 1 (Battery T1) | °C | `rawTemp / 10` -> (°C) |

## 4. 📊 ข้อมูลเรียลไทม์หลัก (Real-Time Telemetry)
* **Register เริ่มต้น:** `0x1290`
* **จำนวนที่อ่าน:** 20 Registers
* **ลักษณะพิเศษ:** บีบอัดข้อมูลแบบไม่มีช่องว่าง (Gaps-Free sequence)

| ตําแหน่งดึงค่า (Index) | Register | ข้อมูลที่ดึงได้ | ขนาด | หน่วยวัด | การคำนวณในโค้ด |
| :---: | :---: | :--- | :---: | :---: | :--- |
| `rtRegs2[0:1]` | `0x1290:0x1291` | แรงดันไฟฟ้ารวม (Total Voltage) | 32-bit (U32) | mV | `mV / 1000` -> (V) |
| `rtRegs2[2:3]` | `0x1294:0x1295` | กำลังวัตต์เรียลไทม์ (Battery Power) | 32-bit (S32) | mW | `mW / 1000` -> (W) |
| `rtRegs2[4:5]` | `0x1298:0x1299` | กระแสไฟรวม (Pack Current) | 32-bit (S32) | mA | `mA / 1000` -> (A) |
| `rtRegs2[11]` | `0x129B` | เปอร์เซ็นต์ประจุคงเหลือ (SOC) | 16-bit (U16) | % | ค่าตรง |

*หมายเหตุ: กระแสไฟและวัตต์เป็นเลขติดลบสำหรับสถานะ "จ่ายไฟ/ดิสชาร์จ" (Discharging) และเลขบวกสำหรับสถานะ "ชาร์จไฟ" (Charging)*
